The aim of the CHILD network is to provide a tool that leverages both clinical expertise and historic data. The CHILD network is well presented in papers on the subject and can be found at (https://scholar.google.com/scholar_url?url=https://projecteuclid.org/journals/statistical-science/volume-8/issue-3/Bayesian-Analysis-in-Expert-Systems/10.1214/ss/1177010888.pdf&hl=en&sa=X&ei=e0mwZYLjMYye6rQPptCIyAY&scisig=AFWwaeYW3xst6U6B46PEcWwH55Go&oi=scholarr).

Here, I have taken the network data and created a DAG (Directed Acyclic Graph) that mirrors the CHILD network. The DAG is useful as a graphical representation used to model probabilistic relationships and dependencies among different variables, offering insight to practitioners. The network represents the incidence and presentation of six possible diseases that would lead to a blue baby. This markdown file highlights some of the steps I took to answer questions related to the Bayesian Network (BN) created using the data associated with the above linked article.

The first step in creating a network is to construct an empty DAG. We must do this before we can determine any joint distributions.

# Step 1: create an empty dag based on network structure
dag <- empty.graph(nodes = c("BirthAsphyxia", "HypDistrib", "HypoxiaInO2", 
                             "CO2", "ChestXray", "Grunting", "LVHreport", "LowerBodyO2",
                             "RUQO2", "CO2Report", "XrayReport", "Disease", "GruntingReport",
                             "Age", "LVH", "DuctFlow", "CardiacMixing", "LungParench", 
                             "LungFlow", "Sick"))

The DAG shows directionality now, but needs to be parameterized. The most useful method for this network is to parameterize using the CHILD data itself. We can easily parameterize using bn.fit. The ISS is the imaginary sample size, for which R will run the specified number of Monte Carlo simulations.

# Parametrize using provided data, estimating conditional probabilities
bn.bayes2 <- bn.fit(dag, data = dats, method = "bayes", iss = 250)

Spieghalter, et al, state that the diseases being evaluated include six possible conditions that are mutually exclusive and exhaustive. The authors state that the level of oxygen in the lower body is directly related to the underlying level when breathing oxygen and that the cardiac mixing also influences hypoxia (page 221). It is important to understand this, as this network is a diagnostic tool, and we are assuming that the child is being evaluated for which disease is the most likely, not the likelihood of overall disease. With this information in mind, I generated a plot about the most likely diagnoses given birth asphyxia. Using an ISS of 250, the highest probability of disease is TGA.

We can also now visualize information on the network. We can use basic plotting (as we did above, which was extremely hard to read) or use a more advanced plot available within graphviz. Using graphviz, we can mimic the network provided in the reference paper.

graphviz.plot(dag)

The visualizations can be highly customized. But more importantly, we can now visualize some of the parameters. Lets take a look at overall probability first.

# Visualize some of the parameters
# Recall that the model is bn.bayes2 <- bn.fit(dag, data = dats, method = "bayes", iss = 250)

bn.fit.barchart(bn.bayes2$Disease, main = "Disease", 
                xlab = "Most Likely Diagnosis | Birth Asphyxia", ylab = "")

We can then set evidence and start to compare changes to the network graph. While the size of this graph is not conducive to an actual diagnostic setting, it still shows the capability of a BN to visualize the changes in probability by variable, which is extremely useful. We can also run queries to show the conditional probabilities of each outcome, which is again extremely useful.

# Take a look at original Bayesian Network with probabilities
# str(dats)

graphviz.chart(bn.bayes2, grid = TRUE, main = "Original BN")

# Set Evidence to feed into the network graph
unique(dats$XrayReport)
## [1] Grd_Glass  Asy/Patchy Normal     Oligaemic  Plethoric 
## Levels: Asy/Patchy Grd_Glass Normal Oligaemic Plethoric
junction <- compile(as.grain(bn.bayes2))
jedu <- setEvidence(junction, evidence = list(LowerBodyO2 = c("5-12", "12+"), XrayReport = "Plethoric"))

# Absorb the information above into a graph
graphviz.chart(as.bn.fit(jedu, including.evidence = TRUE), grid = TRUE,
               main = "BN with Evidence")
